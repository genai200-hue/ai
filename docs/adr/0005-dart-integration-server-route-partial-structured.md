# ADR-0005: DART 실연동 — 서버 라우트 + corp_code 매핑 + 부분 정형/규칙 요약

- 상태: 채택
- 날짜: 2026-07-21
- 관련: ADR-0001(공시 DART 단일 출처, mock 우선), ADR-0003(요약 사실 중립), ADR-0004(공시 단위 요약)

## 맥락

mock 단계를 마치고 OpenDART 실공시를 연동한다. 다음 제약이 있다.

1. **브라우저 직접 호출 불가**: OpenDART는 CORS를 허용하지 않고, 인증키를 클라이언트에
   두면 그대로 노출된다.
2. **조회 키 불일치**: 화면·관심종목은 6자리 종목코드를 쓰지만 DART 공시검색(`list.json`)은
   8자리 고유번호(corp_code)로 필터한다.
3. **정형 수치는 유형별 API에만 존재**: `list.json`은 제목·유형·날짜·접수번호·제출인만 준다.
   실제 금액(취득금액·발행규모 등)은 유형별 정형 API에만 있고, 그마저 일부 유형만 제공한다.
   단일판매·공급계약, 영업(잠정)실적 등은 정형 JSON API가 없다.
4. **요약 생성 수단**: 사실 중립 요약(ADR-0003)을 만들되, 이 단계에서는 LLM을 도입하지 않는다.

## 결정

- **서버 경유**: 공시 조회는 Next.js Route Handler(`app/api/disclosures`)에서 수행하고,
  인증키는 서버 환경변수 `OPENDART_API_KEY`에서만 읽는다. 브라우저는 이 라우트만 호출한다.
- **corp_code 매핑 고정표**: 주요 종목 유니버스(20종목)의 종목코드→corp_code를 `corpCode.xml`에서
  1회 해석해 `lib/dart/corpCodes.ts`에 고정한다. 유니버스가 늘면 파일을 다시 받아 갱신한다.
- **부분 정형 커버리지**: `list.json` + 자기주식취득결정(`tsstkAqDecsn`) + 유상증자결정(`piicDecsn`)의
  정형 값을 접수번호로 결합한다. 정형 API가 없는 유형은 **수치를 만들지 않고** 목록 정보 + 원문
  링크로 처리한다.
- **규칙 기반 사실 요약**: 정형 값과 목록 메타로 사실 중립 서술을 규칙 생성한다(LLM 미사용).
  전망·평가·매매의견은 넣지 않는다(ADR-0003).
- **노이즈 필터**: 임원·주요주주 정기 지분신고 등 대량 반복 공시는 기본 제외해 주요 공시가
  묻히지 않게 한다(`lib/dart/enrich.ts`의 `NOISE_REPORTS`, 조정 가능).

## 결과

- 화면은 데이터 소스 변경과 무관하게 `/api/disclosures` seam만 소비한다.
- 실데이터가 흐르되, 정형 수치는 자기주식취득·유상증자 등 일부 유형에서만 채워진다.
- 키는 커밋/클라이언트 번들 어디에도 노출되지 않는다.

## 한계 / 후속

- **유형 분류는 키워드 기반**이라 오분류 여지가 있다.
- **목록은 종목당 최신 100건 상한**(페이지네이션 미구현) — 매우 활발한 종목은 누락 가능.
- **공급계약·실적의 수치**는 원문(document) 파싱이 필요하며 별도 과제.
- **LLM 요약**은 미도입. 도입 시 별도 키·비용·검증(ADR-0003 준수)이 필요하다.
- 배포는 Vercel. DART가 국내이므로 서버리스 리전 `icn1`(서울) 권장(`vercel.json`).
