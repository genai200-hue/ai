# LLM Wiki

개인 지식베이스를 LLM으로 구축·유지하는 패턴. 이 문서는 자신의 LLM 에이전트에 복사·붙여넣기 하도록 설계된 "아이디어 파일"로, 고수준 아이디어만 전달하고 구현 세부는 사용자와 에이전트에게 맡긴다.

## The Core Idea

LLM·문서 상호작용의 대부분은 **검색(retrieval)** 패턴이다: 파일을 올리면 질의 시점에 관련 청크를 검색해 답을 생성한다. 동작하지만, 매 질문마다 지식을 처음부터 재발견할 뿐 **아무것도 축적되지 않는다.**

이 패턴은 근본적으로 다르다: 원자료에서 매번 검색하는 대신, **LLM이 지속적인 위키 — 구조화되고 상호링크된 마크다운 모음 — 를 점진적으로 구축·유지한다.** 소스를 추가하면 LLM이 읽고, 핵심을 추출하고, 기존 위키에 통합한다 — 엔티티 페이지를 갱신하고, 요약을 개정하고, 모순을 표시한다.

**핵심 차이: 위키는 지속적이고 복리로 쌓이는 산출물이다.** 상호참조가 존재하고, 모순이 표시되며, 종합은 소비한 모든 것을 반영한다. 소스와 질문이 쌓일수록 위키는 더 풍부해진다.

사람은 소스를 큐레이션하고 탐색한다. LLM은 잡일 — 요약·상호참조·정합성 유지 — 을 한다. 개인 지식, 연구, 독서, 비즈니스 팀, 경쟁 분석 등에 두루 통한다.

## Architecture

3계층:

- **Raw sources** — 불변의 큐레이션된 문서(글·논문·이미지·데이터). LLM은 읽되 절대 수정하지 않는다.
- **The wiki** — LLM이 생성한 마크다운(요약·엔티티·개념 페이지). LLM이 이 계층을 전적으로 소유하며 페이지를 만들고 갱신하고 상호참조를 유지한다.
- **The schema** — 위키 구조·규약·워크플로우(ingest/query/maintenance)를 LLM에 알려주는 문서(예: CLAUDE.md). 시간에 따라 LLM과 함께 진화시킨다.

## Operations

- **Ingest** — 소스를 넣고 처리를 요청한다. LLM이 읽고, takeaway를 논의하고, 요약을 쓰고, 인덱스를 갱신하고, 관련 엔티티/개념 페이지를 개정하고, 로그를 추가한다. 한 소스가 10~15개 페이지를 건드릴 수 있다.
- **Query** — 위키에 질문한다. LLM이 관련 페이지를 찾아 인용과 함께 종합한다. 답은 새 위키 페이지로 다시 파일링할 수 있다 — 발견한 비교·분석·연결이 채팅 히스토리로 사라지지 않게.
- **Lint** — 주기적으로 위키를 건강 점검한다: 모순, 오래된 주장, 고아 페이지, 누락된 상호참조, 데이터 공백.

## Indexing and Logging

- **index.md** — 위키 전체의 콘텐츠 지향 카탈로그: 각 페이지를 링크·한 줄 요약·(선택)메타데이터와 함께 카테고리별로 나열. 매 ingest 시 갱신. 질의 시 LLM은 인덱스를 먼저 읽는다.
- **log.md** — append-only 시간순 기록(ingest/query/lint). 각 항목이 일관된 접두사(예: `## [2026-04-02] ingest | Article Title`)로 시작하면 간단한 도구로 파싱 가능해진다.

## Optional: CLI Tools

규모가 커지면 위키 페이지에 대한 검색엔진이 유용해진다. LLM 재랭킹을 곁들인 하이브리드 BM25/벡터 검색을 CLI·MCP 서버로 쓸 수 있다.

## Tips and Tricks

- **Obsidian Web Clipper**로 웹 글을 마크다운으로 변환
- **이미지를 로컬로 다운로드**하면 Obsidian에서 LLM이 보고 참조 가능
- **Obsidian 그래프 뷰**로 위키의 형태(연결·허브·고아)를 확인
- **Marp**로 마크다운 기반 슬라이드 생성
- **Dataview**로 frontmatter를 질의해 동적 테이블 생성
- 위키는 git 저장소 — 버전 이력·협업을 공짜로 얻음

## Why This Works

지식베이스 유지의 고된 부분은 부기(bookkeeping)다: 상호참조 갱신, 요약 최신화, 모순 기록, 정합성 유지. 사람은 유지 부담이 가치보다 빨리 커져 위키를 방치한다. LLM은 지루해하거나 잊지 않는다. 사람은 소스를 큐레이션하고, 분석을 지시하고, 질문하고, 의미를 생각한다. 나머지는 LLM이 처리한다.

정신적으로 Vannevar Bush의 **Memex(1945)** — 문서 간 연상 경로를 지닌 개인적·큐레이션된 지식 저장소 — 와 닮았다. Bush의 비전은 웹이 된 것보다 이쪽에 가까웠다: 사적이고, 능동적으로 큐레이션되며, 연결이 문서만큼 가치 있는. LLM은 Bush가 풀지 못한 유지의 난제를 푼다.

## Note

이 문서는 의도적으로 추상적이다. 구현 세부(디렉터리 구조·스키마 규약·페이지 포맷·툴링)는 도메인·취향·LLM에 따라 다르다. 언급된 모든 것은 선택적·모듈식이다. 이 문서를 LLM과 공유하고 함께 자신에게 맞는 버전으로 인스턴스화하라.
