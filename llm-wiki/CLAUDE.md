# LLM Wiki — 스키마 & 운영 규칙

이 디렉터리는 [Andrej Karpathy의 "LLM Wiki" 패턴](sources/2026-07-21-karpathy-llm-wiki.md)을 따르는 **누적형 지식베이스**입니다. 사람은 소스를 큐레이션하고 질문하며, LLM(=당신)은 요약·상호링크·정합성 유지 같은 잡일을 대신합니다.

> 이 파일은 위키의 **스키마**입니다. 위키에서 작업할 때 항상 먼저 읽고, 규칙이 현실과 안 맞으면 이 파일을 함께 갱신하세요(위키와 co-evolve).

## 3계층 구조

1. **원본 소스** — 불변. 큐레이션된 원자료. **읽기만** 하고 수정하지 않는다. 두 종류가 있다:
   - **외부 소스** → `sources/`에 마크다운으로 취득·저장(예: gist, 논문).
   - **저장소 내부 소스** → 이 프로젝트가 이미 소유한 큐레이션 문서(`CONTEXT.md`, `docs/specs/*`, `docs/adr/*`, 루트 `CLAUDE.md`)와 코드(`lib/*`). **복사하지 않고 제자리에서 상대경로로 참조**한다(단일 진실 유지). 목록은 `index.md`의 Sources 참고.
2. **위키 (`pages/`)** — LLM이 생성·소유하는 마크다운. 요약·개념·엔티티 페이지. 자유롭게 만들고 갱신하며 상호참조를 유지한다.
3. **스키마 (이 파일)** — 위키 구조·규칙·워크플로우. 위키와 함께 진화한다.

## 디렉터리 규약

```
llm-wiki/
├── CLAUDE.md                ← 이 파일 (스키마)
├── index.md                 ← 전체 카탈로그 (매 ingest 시 갱신)
├── log.md                   ← append-only 활동 로그
├── sources/                 ← 원본 소스 (불변)
│   └── YYYY-MM-DD-slug.md
└── pages/
    ├── concepts/            ← 개념 페이지
    ├── entities/            ← 인물·조직·제품 페이지
    └── tools/               ← 도구·기법 페이지
```

## 페이지 규약

모든 `pages/` 문서는 frontmatter로 시작한다:

```yaml
---
title: 사람이 읽는 제목
type: concept | entity | tool
tags: [태그, ...]
sources: [YYYY-MM-DD-slug, ...]   # 이 페이지가 근거한 원본 소스 slug
updated: YYYY-MM-DD
---
```

- **상호참조**: 다른 위키 페이지는 상대경로 마크다운 링크로 연결한다(터미널·GitHub·Obsidian 모두에서 클릭 가능). 예: `[LLM Wiki 패턴](../concepts/llm-wiki-pattern.md)`.
- **Sources 섹션**: 페이지 하단에 근거 소스를 링크로 명시한다. 위키의 모든 주장은 원본 소스로 추적 가능해야 한다.
- **파일명**: kebab-case, 안정적으로 유지(링크가 깨지지 않도록). 이름을 바꾸면 참조도 함께 갱신.
- 한국어로 작성한다(이 저장소 규약).

## 운영 (Operations)

### Ingest — 새 소스 추가
1. 원본을 `sources/YYYY-MM-DD-slug.md`에 저장한다(불변).
2. 핵심을 사용자와 짚은 뒤, 관련 개념/엔티티/도구 페이지를 **새로 만들거나 갱신**한다(한 소스가 보통 10~15개 페이지를 건드림).
3. 새 정보가 기존 페이지와 **모순**되면 해당 페이지에 `> ⚠️ 모순:` 블록으로 표시한다(임의로 지우지 말 것).
4. `index.md`를 갱신한다.
5. `log.md`에 `## [YYYY-MM-DD] ingest | <소스 제목>` 항목을 추가한다.

### Query — 질문에 답하기
1. 먼저 `index.md`를 읽어 관련 페이지를 찾는다.
2. 해당 페이지들을 근거로 **인용과 함께** 답한다.
3. 답이 재사용 가치가 있으면(비교·분석·연결) 새 위키 페이지로 파일링하고 `index.md`·`log.md`에 반영한다.

### Lint — 건강 점검 (주기적)
- 모순, 오래된 주장, 고아 페이지(아무도 링크 안 함), 누락된 상호참조, 근거 없는 주장(Sources 없음)을 찾는다.
- 결과를 `log.md`에 `## [YYYY-MM-DD] lint | <요약>`으로 기록한다.

## 원칙

- **누적**: 매 상호작용이 위키를 더 풍부하게 만든다. 채팅에만 남고 사라지는 통찰이 없게 한다.
- **추적성**: 모든 위키 주장은 원본 소스로 거슬러 올라갈 수 있어야 한다.
- **정합성 우선**: 모순은 지우지 말고 드러내 사용자가 판단하게 한다.
- **git**: 이 위키는 git 저장소의 일부다 — 버전 이력이 곧 감사 추적이다.
